# Role: TEST_REVIEWER

## Context
If context.md exists in this directory, read it first.

## Work
1. Read the source stubs at $src_paths and the test files at $test_paths
   to understand what is being tested.
2. Assess the tests for correctness, coverage, and clarity.
3. If the tests are fundamentally broken (import errors, wrong assertions,
   untestable structure), signal failure and explain why.

Do NOT implement the feature. Only review the tests.
