### What exists but isn't wired

- **`WorkstreamRunner`** — has `prepare()`, `merge()`, `teardown()` methods,
  but the orchestrator doesn't call them. Workstream state transitions
  (`activate`, `deactivate`, `mark_merged`, `mark_failed`) happen inline in
  the orchestrator, but no actual I/O (creating worktrees, merging integration
  PRs, tearing down worktrees) is dispatched.

- **`build_standard_runner()`** — factory that wires all six `StepDispatch`
  tables with default implementations, but nothing calls it yet.

- **Workstream implementations** (`GitWorkstreamPreparer`, `GhWorkstreamMerger`,
  `GitWorkstreamTeardown`) — concrete classes exist, not composed into anything.

### What doesn't exist yet

- **CLI entry point** (`run_graph.py`) — no way to run a graph from the command
  line outside the prototype.
- **Reset tool** (`reset_graph.py`) — no clean architecture version.

## PR plan

### PR N1: Wire WorkstreamRunner into Orchestrator

- Branch: `feat/orchestrator-workstream-runner`

The orchestrator currently manages workstream state transitions inline but
doesn't dispatch any workstream I/O. This PR wires `WorkstreamRunner` into
`Orchestrator`, cleans up redundant mutable state, adds `MERGE_READY` as an
intermediate status for future human-approval gates, and adds a
`fail_fast_on_workstream_error` config option.

**Changes:**

1. Promote `WorkstreamRunner` to a `@runtime_checkable Protocol`; rename the
   concrete implementation to `StandardWorkstreamRunner`.
2. Add `workstream_runner: WorkstreamRunner` field to `Orchestrator`.
3. Before first task in a workstream: call `workstream_runner.prepare()`.
   `prepare()` owns the PENDING → ACTIVE transition.
4. Add `MERGE_READY` status. `_refresh_workstream_terminal_states` transitions
   to MERGE_READY; a new `_process_merge_ready_workstreams` helper calls
   `workstream_runner.merge()` (decouples detection from I/O).
5. After main loop exits: call `workstream_runner.teardown()` for each
   workstream that was prepared.
6. Remove `active_task_id` from `WorkstreamState` and `activate()`/`deactivate()`
   from `WorkstreamRuntime`. Derive "has running task" by scanning task runtimes
   for `RUNNING`/`PR_CREATED` status.
7. Add `fail_fast_on_workstream_error: bool = True` to `OrchestratorConfig` —
   blocks preparing new workstreams after any workstream fails; does not cancel
   in-flight work.

**Acceptance criteria:**
- [x] Orchestrator accepts `workstream_runner` parameter.
- [x] Workstream prepare/merge/teardown are called at correct lifecycle points.
- [x] `MERGE_READY` intermediate status decouples state detection from merge I/O.
- [x] `active_task_id` removed; active-task constraint derived from task statuses.
- [x] `fail_fast_on_workstream_error` config blocks new workstream preparation.
- [x] `WorkstreamRunner` is a Protocol; `StandardWorkstreamRunner` is the impl.
- [x] All existing orchestrator tests pass (with `NoOpWorkstreamRunner` double).
- [x] Six new lifecycle tests added.
- [x] `pixi run check` passes.
