### PR F: Agent-assisted integration branch merging (optional)

- Branch: `feat/agent-merge`

> **Decision gate:** Before starting this PR, evaluate whether to implement
> now or defer to `docs/BACKLOG.md`. The value depends on how often merge
> conflicts arise in practice and whether human review is sufficient.

**Merge strategy options (configurable per-graph or per-workstream):**
- `human` — Create integration PR, leave for human review and merge (default,
  current behavior after PR A2).
- `auto` — If no merge conflicts, merge automatically. If conflicts exist,
  leave for human review.
- `agent` — If merge conflicts, launch an agent to resolve them. Require
  the test suite to pass before completing the merge. If the agent cannot
  resolve, leave for human review.

**Implementation sketch:**
- Add a `merge_strategy` field to WorkstreamSpec or OrchestratorConfig.
- For `auto`: attempt `gh pr merge`, fall back to `human` on failure.
- For `agent`: create a tmux pane with Claude Code, instruct it to resolve
  conflicts and run tests. Agent uses TaskHelper-like API to signal success
  or failure.

**Acceptance criteria:**
- [ ] `human` mode creates PR without merging (same as PR A2)
- [ ] `auto` mode merges when clean, falls back when not
- [ ] `agent` mode resolves conflicts and runs tests before merging
- [ ] `pixi run check` passes
- [ ] E2E: test with conflicting parallel workstreams
