# Sprint Notes — 2026-03-19

> **Status: In Progress.**

## Goal

Systematically build out end-to-end test coverage by category. Each category
corresponds to a subdirectory under `graphs/` with YAML graphs, README docs,
and observations from test runs. Fix bugs and wire unimplemented features as
needed to enable each test category.

## Context

E2E testing infrastructure is in place (e2e_run.sh, e2e_reset.sh, e2e_check.sh).
Smoke tests exist in `graphs/smoke/`. Real-time console output and verbose mode
were added in PR #112. The dispatch race condition was fixed in PR #111 and the
outcome reporting bug in PR #113.

Investigation revealed several features that are scaffolded (parsed from YAML,
written to policies.json) but not wired in the runner: completion gates, review,
and ADR production. A concerns bug was also found (signal.concerns dropped in
the runner).

### PR D: Wire completion gates + gates/ test graphs

- Branch: `feat/completion-gates`

**Wire gate execution:**
- Add gate step to `StandardTaskRunner` (after completion signal, before merge)
- Run `completion_gate` shell command in the task's worktree
- On failure: retry up to `max_gate_attempts`, then mark task failed
- Emit step-level events: `task_gate_running`, `task_gate_passed`/`task_gate_failed`

**Test graphs:**
- `graphs/gates/README.md` — category docs
- Graph with `completion_gate: "pixi run test"` — verifies agent's code passes tests

**Acceptance criteria:**
- [ ] Gate command runs after task signals .done
- [ ] Gate failure triggers retry (up to max_gate_attempts)
- [ ] Gate pass proceeds to PR merge
- [ ] Verbose output shows gate events
- [ ] `pixi run check` passes
- [ ] E2E: gate graph runs, tests pass, PR merges
