# Sprint Notes — 2026-03-25

> **Status: In Progress.**

## Goal

Wire cross-workstream dependency ordering so that tasks depending on work
in other workstreams are not dispatched prematurely. Enable the orchestrator
to detect when integration PRs are merged (initially via GitHub polling),
and optionally auto-merge integration PRs when conditions are met.

## Context

The orchestrator dispatches a task when all its DAG dependencies have
task-level status PR_MERGED (merged into the dependency's workstream
integration branch). This is correct for same-workstream dependencies,
where tasks share an integration branch. But for cross-workstream
dependencies, the upstream workstream's integration PR must also be merged
to main — otherwise the downstream task's worktree (branched from main)
is missing the dependency's code.

During e2e testing (sprint 2026-03-19, `diamond_4_workstreams.yaml`), an
agent in Workstream Y discovered its upstream code was missing, found the
Workstream X integration PR awaiting human review, and merged it
unilaterally. The task succeeded, but the coordination model was violated:
a human-reviewed PR was merged without approval, and the agent didn't
report the workaround. See `docs/discussions/AGENT_ISOLATION_REFINED.md`
for full analysis.

### Architecture notes

**Current dispatch pipeline** (three checkpoints in `orchestrator.py`):

| Function | What it checks |
|---|---|
| `TaskGraph.ready_ids()` | Task-level DAG dependencies (pure graph, no workstream awareness) |
| `_workstream_can_run()` | Workstream state, serialization, parent hierarchy |
| `_blocked_reason()` | Permanent blockers: failed deps, failed/pending workstreams |

The cross-workstream check fits into `_workstream_can_run()` or nearby.

**Integration PR lifecycle:**

1. All tasks in workstream complete → `MERGE_READY`
2. Orchestrator creates integration PR → `PR_CREATED`
3. Human merges PR to main → `MERGED` (currently no detection mechanism)

Step 3 is the gap — the orchestrator has no way to know when the human
has merged the PR. This sprint adds polling-based detection (PR A) and
optional orchestrator-driven merge (PR B).

**Platform coupling:** GitHub-specific code is isolated behind protocols
(`TaskMerger`, `WorkstreamIntegrator`, etc.) with concrete `Gh*`
implementations. The new `IntegrationMergeChecker` protocol follows
the same pattern, keeping the orchestrator platform-agnostic.

### PR B: Orchestrator-driven integration PR merge (optional auto-merge)

- Branch: `feat/auto-merge-integration`

**WorkstreamSpec extension:**
- `auto_merge: bool` field on `WorkstreamSpec` (default: false).
- YAML parsing + validation in `task_graph/builder.py`.

**Auto-merge logic:**
- When a workstream reaches `MERGE_READY` and `auto_merge` is true:
  1. Check all tasks in the workstream for design concerns. If any exist,
     skip auto-merge (fall back to human review).
  2. Create the integration PR (existing behavior).
  3. Merge the integration PR via `gh.pr_merge()`.
  4. Transition workstream to `MERGED`.
- When `auto_merge` is false (default), behavior unchanged — integration
  PR created and left for human review.

**Concern-gated merge:**
- Auto-merge is conditional on zero design concerns across the
  workstream's tasks. If any task recorded a design concern, the
  integration PR is created but NOT auto-merged, regardless of the
  `auto_merge` setting. This ensures human review when agents flag
  potential issues.

**Tests:**
- Unit tests for auto-merge logic (with and without concerns)
- Unit tests for YAML parsing of `auto_merge` field
- Tests verifying concern-gated behavior (concerns present → no auto-merge)

**E2E validation:**
- New graph in `graphs/workstreams/` with `auto_merge: true` — verify
  the integration PR is merged automatically when no concerns exist.
- Variant where an agent records a concern — verify auto-merge is skipped.

**Acceptance criteria:**
- [ ] `auto_merge: true` causes orchestrator to merge integration PR
- [ ] Auto-merge skipped when design concerns exist
- [ ] `auto_merge: false` (default) preserves current human-review behavior
- [ ] `pixi run check` passes
- [ ] E2E: auto-merge graph completes without human intervention
