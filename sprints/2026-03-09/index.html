    Sprint Notes — 2026-03-09

    Phase 2 plan: original item 6 (adapter layer + real runner)

    External testbed assumptions

    1. External repo for manual integration/E2E work: /data/git/agentrelaydemos/main/.
    2. New-style YAML graphs for non-prototype runner tests will live under /data/git/agentrelaydemos/main/graphs/.
    3. Old prototype graph files remain archived under /data/git/agentrelaydemos/main/graphs/archived/.
    4. During early PRs, it is acceptable to use synthetic temp repos for deterministic tests and use agentrelaydemos for manual smoke checks.

    PR I: integration contracts + error model

    • Branch: feat/integration-contracts
    • Scope:
    • Define non-prototype adapter interfaces for workspace, signal I/O, PR ops, and agent launch boundaries.
    • Define typed exceptions and error classification mapping (expected task failure vs internal/system failure).
    • Document invariants in architecture/diagram.
    • Acceptance-test checklist:
    • [x] New contracts compile cleanly (pixi run typecheck).
    • [x] Unit tests cover classification and exception mapping behavior.
    • [x] No behavioral changes to existing task/orchestrator tests.

    PR I implementation plan (detailed)

    1. Create a new integration package (contracts only)
    2. Add:
      • src/agentrelay/integration_contracts/__init__.py
      • src/agentrelay/integration_errors/__init__.py

    3. Purpose: define stable side-effect boundaries without changing runtime behavior.

    4. Define adapter contracts (Protocol interfaces)

    5. In integration_contracts, define protocols for four boundaries:
      • WorkspaceAdapter (generalized from local worktree-specific naming)
      • SignalAdapter
      • PullRequestAdapter
      • AgentLauncher
    6. Define generalized workspace refs for local/remote execution:
      • LocalWorkspaceRef
      • RemoteWorkspaceRef
      • WorkspaceRef (union of local/remote refs)

    7. Add IntegrationAdapters dataclass to bundle boundaries for later TaskRunnerIO composition (PR N).

    8. Define typed error model + classification

    9. In integration_errors, define:
      • IntegrationError (base)
      • Boundary-specific subclasses (WorkspaceIntegrationError, SignalIntegrationError, PullRequestIntegrationError, AgentLaunchIntegrationError)
      • Compatibility alias WorktreeIntegrationError (transitional)
      • ExpectedTaskFailureError vs InternalIntegrationError
      • classify_integration_error(exc) returning IntegrationFailureClass

    10. Goal: make expected task failure vs internal/system failure explicit before concrete adapters.

    11. Keep behavior unchanged in existing runtime code

    12. No real wiring into TaskRunner/Orchestrator in PR I.

    13. This PR remains contract-and-model only so downstream adapter PRs can implement against a stable API.

    14. Docs + diagram updates

    15. Update:

      • docs/ARCHITECTURE.md (integration boundary layer)
      • docs/DIAGRAM.md (integration package/module block)
      • API docs/nav entries for integration package as needed.

    16. Tests + validation

    17. Add unit tests:
      • test/test_integration_errors.py
      • test/test_integration_contracts.py
    18. Cover:
      • error class hierarchy
      • classification helper behavior
      • immutability/shape of contract dataclasses
    19. Run:
      • pixi run pytest test/test_integration_errors.py test/test_integration_contracts.py
      • pixi run typecheck
      • pixi run docs-build

    PR J: git/worktree adapter

    • Branch: feat/git-worktree-adapter
    • Scope:
    • Implement git/worktree operations behind adapter interface.
    • Support worktree create/switch/delete and branch setup primitives needed by TaskRunnerIO.
    • Acceptance-test checklist:
    • [ ] Adapter tested against temp git repo fixtures.
    • [ ] Idempotent cleanup behavior covered by tests.
    • [ ] Error mapping for git command failures is deterministic.

    PR K: signal adapter

    • Branch: feat/signal-adapter
    • Scope:
    • Implement instruction write + completion polling + timeout handling.
    • Parse completion signals into TaskCompletionSignal with strict validation.
    • Acceptance-test checklist:
    • [ ] Tests cover done, failed, timeout, malformed payload, missing fields.
    • [ ] Poll interval/timeout behavior is configurable and documented.
    • [ ] No busy-loop behavior under normal polling cadence.

    PR L: tmux launcher + real TmuxAgent

    • Branch: feat/tmux-agent-implementation
    • Scope:
    • Implement TmuxAgent.from_config(...).
    • Implement TmuxAgent.send_kickoff(...).
    • Add readiness/command-injection behavior required by current workflow.
    • Acceptance-test checklist:
    • [ ] Unit tests for tmux command construction and argument handling.
    • [ ] Failure modes (missing tmux session, command failure) raise typed errors.
    • [ ] Optional local smoke command (env-flag gated) documented.

    PR M: PR operations adapter

    • Branch: feat/pr-ops-adapter
    • Scope:
    • Implement PR create/merge/status boundary (likely gh-backed).
    • Normalize command failures into typed adapter errors.
    • Acceptance-test checklist:
    • [ ] Unit tests with subprocess command doubles.
    • [ ] Merge failure paths map to expected failure vs internal error per contract.
    • [ ] Adapter API is framework/environment-agnostic.

    PR N: concrete TaskRunnerIO composition

    • Branch: feat/task-runner-io-concrete
    • Scope:
    • Compose adapters from PRs J/K/L/M into a concrete TaskRunnerIO.
    • Wire task runtime fields (worktree path, branch, agent, PR URL, errors, concerns).
    • Acceptance-test checklist:
    • [ ] TaskRunner integration tests run against concrete TaskRunnerIO using fakes/temp dirs.
    • [ ] Teardown modes (always, on_success, never) behave as expected.
    • [ ] Task failure and internal-error boundaries remain consistent with PR G policy.

    PR O: non-prototype runner entrypoint

    • Branch: feat/nonprototype-run-graph-cli
    • Scope:
    • Add non-prototype run_graph executable/module.
    • Support external graph paths (including agentrelaydemos), runtime config flags, and orchestrator wiring.
    • Acceptance-test checklist:
    • [ ] CLI can parse and run new-style graph YAML via TaskGraphBuilder.from_yaml(...).
    • [ ] CLI supports configurable teardown/retry/concurrency flags.
    • [ ] Dry-run/validation-only mode exists for quick schema checks.

    PR P: non-prototype reset tool

    • Branch: feat/nonprototype-reset-graph
    • Scope:
    • Implement a clean non-prototype reset_graph command for run cleanup.
    • Reset artifacts needed by new flow (worktrees, branches, signal/run-state files) safely and predictably.
    • Keep prototype reset tooling separate and unchanged.
    • Acceptance-test checklist:
    • [ ] Reset is idempotent when rerun.
    • [ ] --yes (non-interactive) and interactive confirmation both supported.
    • [ ] Cleanup verified against local sandbox repo and manual agentrelaydemos smoke.

    PR Q: E2E docs/examples + migration hardening

    • Branch: feat/e2e-runbook-and-graphs
    • Scope:
    • Add new-style E2E graph examples for /data/git/agentrelaydemos/main/graphs/.
    • Document full run + reset workflow for non-prototype stack.
    • Capture known operational caveats and troubleshooting notes.
    • Acceptance-test checklist:
    • [ ] At least one documented end-to-end run/cleanup cycle succeeds on agentrelaydemos.
    • [ ] Docs point users away from graphs/archived/ for new runner tests.
    • [ ] Workflow, guide, and migration docs are mutually consistent.

    Merge discipline for phase 2

    1. Complete each PR and merge to main before starting the next PR.
    2. Keep each PR vertically scoped with runnable/testable acceptance criteria.
    3. Do not collapse PR P (reset tooling) into PR O; reset ergonomics are first-class operational requirements.
