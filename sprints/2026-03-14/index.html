    Sprint Notes — 2026-03-14

    Status: Planning.

    Goal

    Complete the end-to-end wiring from graph YAML to running orchestrator. All protocol implementations exist; the remaining work is composition, CLI, and cleanup.

    Context

    The 2026-03-13 sprint built out all protocol implementations and the agent communication protocol. An unplanned architectural improvement (PR #99) then replaced TaskRunnerIO with per-step StepDispatch[T] dispatch tables on a new StandardTaskRunner, promoted TaskRunner to a protocol, and moved workstream context (integration_branch, workstream_worktree_path) from implementation constructors into TaskState.

    #99 StandardTaskRunner with StepDispatch (unplanned) 2026-03-14

    What exists but isn't wired

    • WorkstreamRunner — has prepare(), merge(), teardown() methods, but the orchestrator doesn't call them. Workstream state transitions (activate, deactivate, mark_merged, mark_failed) happen inline in the orchestrator, but no actual I/O (creating worktrees, merging integration PRs, tearing down worktrees) is dispatched.

    • build_standard_runner() — factory that wires all six StepDispatch tables with default implementations, but nothing calls it yet.

    • Workstream implementations (GitWorkstreamPreparer, GhWorkstreamMerger, GitWorkstreamTeardown) — concrete classes exist, not composed into anything.

    What doesn't exist yet

    • CLI entry point (run_graph.py) — no way to run a graph from the command line outside the prototype.
    • Reset tool (reset_graph.py) — no clean architecture version.

    PR plan

    PR N1: Wire WorkstreamRunner into Orchestrator

    • Branch: feat/orchestrator-workstream-runner

    The orchestrator currently manages workstream state transitions inline but doesn't dispatch any workstream I/O. This PR adds WorkstreamRunner as a dependency of Orchestrator and inserts lifecycle calls at the right points in the scheduling loop.

    Changes:

    1. Add workstream_runner: WorkstreamRunner field to Orchestrator.
    2. Before first task in a workstream: call workstream_runner.prepare(ws_runtime).
    3. After all tasks in a workstream succeed (when mark_merged() fires): call workstream_runner.merge(ws_runtime).
    4. At orchestration end (or on failure): call workstream_runner.teardown(ws_runtime) for each workstream that was prepared.

    The existing workstream state machine (activate, deactivate, mark_merged, mark_failed) stays — those track scheduling state. The runner calls are the actual I/O operations.

    Acceptance criteria: - [ ] Orchestrator accepts workstream_runner parameter. - [ ] Workstream prepare/merge/teardown are called at correct lifecycle points. - [ ] All existing orchestrator tests pass (with mock workstream runner). - [ ] pixi run check passes.

    PR N2: Composition and CLI entry point

    • Branch: feat/run-graph-cli
    • New module: src/agentrelay/run_graph.py

    Wire everything together and provide a CLI.

    Scope: 1. Top-level composition function that builds Orchestrator, StandardTaskRunner (via build_standard_runner()), and WorkstreamRunner (via build_workstream_runner() — new builder, mirrors the task runner pattern) from a parsed graph YAML. 2. CLI via python -m agentrelay.run_graph <graph.yaml> with flags: --max-concurrency, --max-task-attempts, --teardown-mode, --tmux-session, --model, --dry-run.

    Acceptance criteria: - [ ] --dry-run validates graph and prints execution plan. - [ ] Integration test verifies full wiring from graph YAML to orchestrator. - [ ] pixi run check passes.

    PR O: Reset tool

    • Branch: feat/reset-graph

    Clean reset command: close PRs, delete branches/worktrees, remove signal dirs. Idempotent, interactive by default with --yes override.

    Acceptance criteria: - [ ] Idempotent when re-run. - [ ] Tested against temp git repo fixture. - [ ] pixi run check passes.

    Acceptance criteria: - [ ] Documented end-to-end run/reset cycle. - [ ] Docs internally consistent. - [ ] pixi run check passes.
