    integration_contracts

    Integration boundary contracts for non-prototype side effects.

    This package defines protocol interfaces for external systems used by task execution. These contracts intentionally describe semantic behavior only and do not define concrete file names or directory layouts.

    CompletionSignal dataclass

    Semantic completion payload observed from a signal boundary.

    Terminal outcome marker from external task execution. "done" means task work finished and should have a mergeable PR. "failed" means task work failed before successful completion.

    Pull request URL for a successful completion path, if available.
    error Optional[str]

    Failure detail for a failed completion path, if available.
    concerns tuple[str, ...]

    Optional semantic concerns captured during execution.

class IntegrationAdapters:
